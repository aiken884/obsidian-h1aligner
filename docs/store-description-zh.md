# 中文簡短描述（商店 / 社群貼文用）

H1Aligner 會自動把 Obsidian 筆記的檔名，對齊到第一個 H1 標題。你只要寫好 `# 標題`，檔案就會跟著變成同樣的名稱，連結不會壞，也不會突然跳出對話框打斷你。

它只做一件事，但做得很徹底：永遠以第一行 H1 為唯一真相（H1 → 檔名，而不是反過來），搭配五種觸發模式、完整安全保護鏈、乾跑預覽與 20 階段復原，讓你可以放心地把一整個 vault 的檔名整理乾淨，再也不用跟 `Untitled 27.md` 共存。

## 觸發模式說明（文件 / 貼文用）

H1Aligner 讓你自己決定「什麼時候」允許檔名被改動，目前有五種模式可以選：

- **開檔時** — 一打開筆記就檢查並改名。很適合在日常使用中順手整理舊 vault，不用多按任何按鍵。
- **編輯後** — 你停筆幾秒鐘之後才改名。先讓你專心打字，等你稍微休息一下時，檔名才會默默跟上新的 H1。
- **兩者皆啟用** — 開啟筆記與編輯 H1 都會觸發改名。適合想要標題與檔名「隨時保持一致」的使用者。
- **切離筆記時** — 只會在你離開當前筆記的一瞬間，把「剛離開的那一篇」改名，永遠不會動到你正盯著看的檔案。這是體感最溫和的選項，不會讓分頁列表突然跳來跳去。
- **僅手動** — 完全不自動改名，所有動作都靠你自己從指令面板或快捷鍵觸發。適合想用 H1Aligner 的安全機制與預覽工具，但希望每一次改名都親自決定的人。
